项目一｜擎舵 3.0：营销创意 Agent 生产质量升级

项目定位：百度营销体系下的 AI 创意生产平台，承接广告图、营销视频、口播素材和批量投放物料生产。

原问题：2.0 是工具集合，用户需要自己找工具、填参数、判断结果、寻找续改入口。它能承载生成能力，但不天然承接营销任务。

我的角色：主导图片 Agent 2.0 整体体验方案，负责 3.0 首页、超级输入框、导航、AI 创编大师 Workflow 及复杂状态原型验证。正确边界是“主导体验方案并协同产研落地”，不是“训练模型”或“独立开发正式产品”。

核心设计判断：从工具集合到 Agent 生产系统，让广告素材从“能生成”走向“可交付”。

平台搭建：从找工具到表达任务

  • 首页升级：从工具货架转为创作启动台，动作包括超级输入框、智能标签、灵感案例、高频入口、任务启动。
  • 导航与空间升级：低频入口收敛到极窄侧导，主空间承接输入、生成结果和后续动作。
  • 任务页升级：结果页不是终点，而是续改和复用枢纽，承接再生成、去编辑器、智能扩图、智能抠图、图转视频、保存资产。

转场：平台解决启动，质量决定交付

平台框架解决怎么开始和怎么连续创作，但真实投放场景还要回答“结果能不能用”。因此第二阶段进入生产质量提质。

六类生产质量约束

内容准确、主体一致、行业适配、营销表达、合规风控、续改效率。

设计价值：我的工作不是直接训练模型，而是把运营侧不可用反馈转译为 Agent 可理解、系统可检查、用户可续改的体验机制。

擎舵 3.0｜图生营销图 Skill：从模糊输入到可执行 Prompt

用户模糊输入：“帮我做得高级点，更有营销感。”

Agent 结构化补全：行业、主体、保留边界、参考图关系、风格、构图、标题、卖点、CTA、Logo 区、禁用内容、风险约束。

流程：输入理解 → 场景路由 → 分场景解析 → 意图补全 → Prompt 生成 → 输出校验 → Image 2 执行。

结论：Skill 的价值不是多一个按钮，而是把广告生产经验转成 Agent 可执行的方法包。

视频流程：目标表达 → 需求澄清 → 创意方案 → 剧本 → 分镜 → 成片设置 → 合成 → 续改与二创。

LUI + GUI + Workflow：纯 LUI 降低启动门槛，但无法承接结构化编辑、结果预览、分镜控制和资产复用；纯 GUI 可控但启动成本高。因此用语言交互承接目标表达，用图形界面承接编辑控制，用 Workflow 承接状态记录、回退和续改。

Vibe Coding：用可运行原型验证多阶段任务状态、局部重生成、中途退出、Agent 加载与失败，不替代研发，只降低复杂 AI 工作流的沟通成本。

项目二｜巧匠平台：AI 生图生产平台，从模型训练到风格资产化

项目定位：基于 LoRA 训练能力，搭建面向广告设计团队的风格模型生产闭环，让 AI 生图从随机出图走向风格可控、批量可用、资产可复用。

核心人设：懂 LoRA 训练闭环的 AI 创意生产体验设计师。

写法顺序：技术代际 → 平台框架 → 模型特质 → 设计参与 → 数据结果 → 复盘。

边界：不要写成我训练了 LoRA 算法，也不要写成所有质量提升都由设计独立带来。准确表达是：模型能力、样本质量、运营策略和产品体验共同影响结果；我的设计价值是让 LoRA 能力被设计团队稳定使用、调用、评估和沉淀。

平台框架：模型资产工作台

  • 训练中心：承接模型名称、基础模型、触发词、训练参数和上传样本。
  • 模型资产：用模型卡、分类、标签、示例图和发布状态，让模型成为团队共享资产。
  • 生图工作台：让训练后的 LoRA 在文生图、批量生图和业务场景中被稳定调用。
  • 质量评估：把模型结果从主观好看，转为可对比、可记录、可复盘的质量判断。

巧匠｜设计参与训练质量

样本准备：训练质量从样本开始

将样本数量、清晰度、风格一致性、主体一致、低质 / 水印 / 文字干扰提醒、训练前检查和样本状态分类转译成用户可理解的训练前规则。

训练过程：训练不是等待，而是可理解的任务状态

设计上传样本、配置参数、提交训练、排队中、训练中、训练完成、训练失败、失败原因、测试生成入口、发布共享。

模型资产：训练完成不是终点，成为资产才有生产价值

通过模型卡片、基础模型、触发词、示例图、风格标签、行业标签、适用场景、团队共享、优质案例关联，让个人训练经验沉淀为团队可复用的生产能力。

生图调用与质量评估

在文生图 / 批量生图中选择模型，设置 Prompt、强度、尺寸和批量生成参数，支持历史模型复用，并用图片质量、真实性、主体元素、Prompt 控制力、风格一致性、广告可用性、复用价值和用户满意度判断模型是否可用。
