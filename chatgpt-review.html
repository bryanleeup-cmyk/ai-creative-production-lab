三、项目一：擎舵 3.0｜营销创意 Agent 生产质量升级

核心叙事：从工具集合到 Agent 生产系统，让广告素材从“能生成”走向“可交付”。重点不是平台改版，也不是 AI 聊天框，而是 Agent + Workflow + LUI + GUI 如何承接营销创意生产任务，并提升图片、视频素材的生产质量。

03 项目 30 秒卡

擎舵是百度营销体系下的人工智能创意生产平台，承接广告图、营销视频、口播素材和批量投放物料生产。2.0 已经具备一键出图、一键成片、AI 模特、创编大师、数字人、专业编辑器、公共资产库等能力，但整体以工具集合呈现，用户需要自己判断工具、填参数、判断结果、寻找续改入口。我的角色是主导图片智能体 2.0 整体体验方案，负责 3.0 首页、超级输入框、灵感、导航和创编大师交互侧工作流。

04-09 第一章：平台搭建

  • 业务背景：2.0 是能力集合，不是任务系统。能力越多，用户越需要判断先用哪个工具、填什么参数、结果不满意后去哪继续改。
  • 设计判断：平台要从“找工具”转向“表达任务”。用户从营销目标开始，而不是从工具名称开始。
  • 首页升级：从工具陈列到创作启动页。超级输入框承接自然语言需求，灵感内容降低从空白开始的启动压力。
  • 导航与空间升级：将低频入口收敛到极窄侧导，释放创作区，让输入、生成结果和后续动作处在同一任务上下文。
  • 任务页升级：生成结果页不再只是结果展示，而是续改与复用枢纽，承接再次生成、去编辑器、智能扩图、智能抠图、图转视频、保存资产等动作。
  • 转场页：平台解决启动，质量决定交付。平台搭建回答“怎么开始”，生产质量机制回答“结果能不能投放、能不能修改、能不能复用”。

运营侧测试显示，“生成出来”不等于“可以投放”。主要不可用问题包括：文字乱码 / 文案重复、上传商品与生成主体不一致、人物 AI 感重、底图精细度不足、生成多余文字、版式单一、无中生有卖点、行业风格不适配、衍生违禁词导致审核风险。

我将问题抽象为六类生产质量约束：内容准确、主体一致、行业适配、营销表达、合规风控、续改效率。对应设计动作是：把模糊输入转译为模型可执行任务，把输入理解、场景路由、分场景解析、意图补全、Prompt 生成、输出校验和图像模型执行组织成可解释的 Skill 工作流。

Agent 介入边界：明确输入时做轻量指令增强；输入模糊、场景复杂或涉及行业规则时进入深度 Skill 解析。图片结果页把“不满意”转化为下一步生产动作，支持扩图、抠图、再生成、去编辑器、图转视频、保存复用等。

15-18 第三章：视频 Agent 过程控制

视频创作不能一次性黑盒生成，因为视频生产成本更高，且涉及脚本、角色、分镜、字幕、口播、Logo、比例、配乐、合成等多个可错节点。设计策略是将视频 Agent 拆成可确认、可编辑、可回退的 Workflow：目标表达 → 需求澄清 → 创意方案 → 剧本 → 分镜 → 成片设置 → 合成 → 续改与二创。

  • 方案：先确认创意方向，避免直接进入高成本生成。
  • 剧本：把卖点、口播、场景和节奏组织成可讨论文本。
  • 分镜：把视频拆成可替换、可重生成、可预览的镜头。
  • 成片设置：集中处理字幕、配乐、Logo、比例等全局项。

LUI + GUI + Workflow 的关系：语言交互负责目标表达、意图解析、需求澄清和阶段推进；图形界面负责结构化编辑、结果控制和可视化续改；工作流负责拆解复杂阶段、记录任务状态、保留上下文和支持回退。

Vibe Coding 原型验证用于验证复杂 Agent 状态：包括首页输入后进入任务、需求澄清、角色生成、剧本生成、分镜生成、局部重生成、成片设置、中途退出保留进度、合成中和完成状态。边界表述：这是用于复杂状态和产研对齐的可运行原型，不是正式产品开发。

19-21 第四章：设计能力沉淀

技术转译表说明我如何把 AI 技术转成体验机制：Agent 从工具调用变成任务推进；Skill 沉淀行业、营销、合规规则；Prompt 将模糊输入转成模型可执行任务；Workflow 拆解复杂创作阶段；LUI 承接自然语言目标；GUI 承接结构化编辑；Visual QA 做生成前检查；Vibe Coding 做状态机验证。

体验指标建议只保留与设计相关的数据：创作启动率、首次任务启动耗时、开始生成路径、结构化信息补全率、用户二次补充率、结果续改率、保存率、资产复用率、视频 Agent 链路完成率、分镜确认率、成片设置完成率、中途退出后恢复率。不要堆 PV、UV、生成量、消费金额等偏运营数据。

项目复盘：我的价值不是训练模型，而是把运营和投放现场的质量问题，转化为用户可表达、Agent 可理解、系统可检查、结果可继续优化的产品体验机制。

四、项目二：巧匠平台｜AI 生图生产平台：从模型训练到风格资产化

核心叙事：基于 LoRA 训练能力，搭建面向广告设计团队的风格模型生产闭环，让 AI 生图从随机出图走向风格可控、批量可用、资产可复用。这个项目要证明候选人不仅懂平台搭建，也懂 LoRA 训练链路如何影响生成质量。

22-23 巧匠项目封面与 30 秒卡

巧匠是面向广告设计团队的 AI 生图生产平台，承接 LoRA 风格模型训练、调用、质量评估与资产复用。技术背景处在 LoRA 训练 + AI 生图阶段，核心不是任务对话推进，而是模型资产的训练、管理和批量调用。我的角色是负责核心交互设计，参与训练链路、模型资产、调用生图、首页工作台和质量评估闭环的体验方案。

24 技术代际：为什么这个阶段需要平台工作台

不同 AI 技术代际，对应不同交互框架。LoRA 训练 + AI 生图阶段的核心对象是模型资产，因此适合平台工作台框架，承接训练、调用、评估和团队复用。Agent + Workflow 阶段的核心对象是目标表达、多阶段任务推进和持续续改，因此适合任务中心框架。不能简单写成顶部导航不好、侧导天然高级，而要写成根据技术背景和用户任务选择最适合的交互框架。

25-27 业务痛点、用户分层和问题定义

  • AI 生图快，但广告生产不够稳：广告物料生产不是“出一张图”就结束，而是要稳定满足风格、批量、质量和复用要求。
  • 大品牌客户：需要专属视觉、品牌一致、风格可控，通用素材库难以满足差异化诉求。
  • 中小客户：需要快速批量产出同风格素材，普通 AI 生图随机性高，复用成本高。
  • 设计问题定义：用户缺少的不是训练入口，而是从样本准备、训练解释、模型测试、模型发布、批量调用到质量评估的完整生产链路。

28-34 平台框架与关键链路

  • 平台框架：训练、调用、评估、沉淀一体化。平台工作台围绕模型资产组织，而不是围绕单个功能入口堆叠。
  • 样本准备：把训练要求转译为用户规则。解释训练类型、样本数量、清晰度、主体一致、风格统一、背景干扰和关键参数含义。
  • 训练过程：让模型训练可感知。展示上传样本、配置参数、提交训练、训练中、训练完成、调用测试、评估结果、发布共享等状态。
  • 模型资产：让训练结果可管理、可共享。模型卡展示名称、基础模型、触发词、示例图、标签和适用场景；发布共享后进入团队可访问空间。
  • 生图调用：让 LoRA 进入广告生产前台。用户在文生图和批量生图中选择模型，设置 Prompt、强度、尺寸和批量生成参数。
  • 质量评估：用数据判断模型是否可用。围绕图片质量、真实性、主体元素和 Prompt 控制力，把生图结果转为可评估标准。
  • 首页改版：从入口集合到生产工作台。首页承担模型资产分发和生产启动，让训练好的模型更高频地进入广告生产。
